# Evidence Rules for Prism Pilot

## Claim controls
- Every material claim must map to a source file and section.
- If claim support is partial, mark it as provisional and keep a pending placeholder.
- Contradictory values across sources must be flagged for adjudication.

## Writing controls
- Keep methods short in the main body.
- Avoid promotional language.
- Maintain consistency in site names, institution names, and acronyms.

## Status labels (required)
- `[COMPLETED EVIDENCE]`
- `[PENDING INPUT]`
- `[INTERPRETIVE SYNTHESIS]`
- `[RECOMMENDATION]`
- `[ASSUMPTION]`

## EOSDA and monitoring controls
- Do not state EOSDA monitoring as fully operational without explicit setup and reporting evidence.
- Keep implementation claims tied to available artifacts only.
